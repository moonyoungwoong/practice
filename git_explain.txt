git hub 명령어 정리

git config --global user.email "myheroes2@gmail.com"  : 내 계정 등록
gif config --global user.name "문영웅" : 이름 등록
git init : 처음 사용할 때 써야함(visual studio 터미널에서)
git add app.txt : 작성한 파일을 깃에 업로드 하고 싶을 때 써야함(staging area에 저장됨)
git commit -m "처음 만들었음: : 작성된 파일을 어떤 버전으로 만들었는지 메시지를 남겨두고 싶을때 써야함(repository에 저장됨)
git add app.txt app2.txt : app.txt파일과 app2.txt 파일 두개를 깃에 업로드하고싶을때 사용
git add . : 폴더 내에 있는 모든 파일을 깃에 업로드하고싶을때 사용
git status : 현재 내가 어떤 파일들을 staging 해 두었는지 궁금할 때 사용
git add&commit을 동시에 가능한 명령어도 있음(아직 모름)
git log --all --oneline : commit 내역 조회
git branch coupon : coupon 이라는 이름의 baanch를 하나 생성 - 기존에 있던 작업에서 무언가를 추가하고싶을때 원본파일에 작성하는것이 아니라 복사본에 작업해서 실행해본다고 생각하면 됨ㅎ
git switch coupon : 생성한 coupon이라는 이름의 branch로 이동 
git branch -M main : 기본 branch 이름이 main으로 변경(github에서는 main 브랜치 이름을 main으로 설정하라고 이야기함)
git push -u 저장소주소 main : 로컬 저장소에 저장되어있던것을 원격저장소(github)에 저장가능
git remote add 변수이름 주소 : git push -u 저장소주소라고 했을 때 저장소주소가 너무 길어서 비효율적이므로 변수에 넣어주는 행위임.
                             (ex git remote add origin https://github.com/moonyoungwoong/practice.git)라고 하고 origin을 불러오면됨) but git push 만 입력해도 됨.
